discover_inventories()[source]¶

Look up intersphinx inventories to offer as autocomplete suggestions.

This method only needs to be called once per application instance.

Assuming that the application has the intersphinx extension enabled and configured, this will look for the intersphinx_named_inventory attribute on the Sphinx application’s build environment and index each of the availble projects.

This will also loop through each domain object and construct the target_types dictionary - very similar to discover_roles() on the Roles feature, except that intersphinx always uses a domain’s namespace even if primary_domain is set in the app’s config.

Finally, as target completions for other projects aren’t going to change over the course of an editing session, it’s also safe to index them once on init.
